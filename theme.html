Installing An HTML5 Theme

Introduction

While Turbo offers several built-in themes to use in your projects, you may want to install your own theme. This tutorial will show you how you can download and install a theme from HTML5 UP and use it in your project.

This tutorial assumes that you have already set up a static Turbo project.

Visit HTML5 UP and download a theme that you'd like to use. For this tutorial, we will use the Landed theme.

After you have unzipped the theme, you will want to open up two finder windows - one for your project and one for the theme. Navigate to the assets folder of your project: Project > Assets. Inside, you will see a number of folders that contain the assets used for styling your application. By default, Turbo comes with a selection of tools built in.

With the two windows open, go to each folder in the theme's assets folder and copy the contents to your project. For example, open the theme's CSS folder, copy the CSS files there, and paste them in your project's CSS folder. This should be relatively straight forward but there are some important caveats to make note of.

In the theme's assets folder, notice that there is a folder for fonts. Our project folder does not have a fonts folder by default so you must copy the fonts folder from the theme over to the project.

Also notice that the theme has a folder called images in its root directory, but our project does not. In a Turbo project, the images folder is contained within the assets folder, not the root directory. So we must copy the images from the theme's images folder and paste them in our images folder which is contained within the assets folder.

Now we need to import our files into our Gulp file. Each Turbo project comes with a Gulp file with some default configurations. If you want to learn more about why you should use Gulp or what you can do with it, check out this article. You will see a number of Gulp tasks already written that do different things for the project. The first one to make note of is the task called 'css.' This task copies our CSS files, minifies them, concatenates them, and then outputs them to a specified destination. Notice that the destination specified for our CSS files is /dist/css. When Gulp runs, it takes our CSS files, applies the above-mentioned rules, and then puts them into the dist folder, which contains another folder called css inside. When your application is running, this is where the CSS is actually coming from. We need to list all of the CSS files that we are using in this task so that those stylesheets will actually be usable in our project.

Open up the unzipped theme in your text editor and go to the index.html page. At the top of your page where the CSS files are imported, note the order. That is the order you will want to use when you list the files in your Gulp file. Import the files in the order that you see them:

		
		'./assets/css/bootstrap.css',
		'./assets/css/main.css',
		'./assets/css/ie9.css',
		'./assets/css/ie8.css',
		'./assets/css/font-awesome.min.css',

Now that we have all of our Gulp tasks set, we need to make an edit so that our fonts will work properly. Go to the main.css file in your assets directory. On line 2, you should see an import statement for the fonts. Either delete or comment out this line.

The next step is to get our theme code into our index.html file. If we haven't made any mistakes, our theme should show up when we run our devserver and it should look almost identical to the design on HTML5 UP. Note that the theme downloads do not include all photo assets that are shown on the demo site so your page will not look 100% like it. HTML5 UP usually includes placeholder images in their theme downloads instead. We also will need to adjust the file paths for the images as well, so until we do that, we will not see any images at all.

The easiest way to get the code in there is to select everything within the body of the theme's index.html page and paste it in the body of the project. While you can also copy/paste the entire index.html page from the theme, you will need to remember to adjust both the CSS and JS paths in the head and at the bottom of the pages.

If you copy/paste just what is in the body, make sure you remember to copy the class that is applied to the body in the theme.

Run the devserver in the root directory of the project by using the command:


$ turbo devserver

If everything worked properly, your theme should show up!

But you will notice that we need to make some adjustments as the images don't show up. As mentioned earlier, our Gulp file packages all of our assets up for us and puts them in the dist folder. So we need to point all of our images to that directory and specifically to the images folder within it.

After you change all of the img paths, make sure you check to see if any href paths point to images. In this theme, we fortunately do not have that issue but it may occur in other themes.

If you refresh your page, you should now see the images have loaded.

Again, because the real images are not included, the theme will not look 100% like the demo site. You will need to supply your own images and if you do need some stock photos, you can use Unsplash, which is actually where HTML5 UP gets its stock photos. Unsplash is a great resource for high-quality, royalty-free images.

Additionally, the theme code may not have exactly the same elements that are used in the demo so don't worry if your project doesn't look exactly like the demo. Some of the themes include an elements page so you can see exactly what built-in elements are included.

If you look at the demo theme, you will see a Layouts tab that shows different layouts for the theme. For this tutorial, we have only covered integrating the main index.html page but it is simple to include these other layouts. If you have different pages in your site or app, you can copy/paste the HTML for one of the layouts into that page. Change the file path references for the images and that should cover it. There is no need to modify your Gulp file. If you want to use one of the alternate layouts instead of the default one, simply copy/paste the HTML for that layout into index.html and change the file paths. That's it.

Some themes, like Landed, include many elements that you can use for building your site or app. Others only include one page, which might be better for a landing page or a portfolio or a site for a small company or startup. Every HTML5 UP theme has a demo that you can play around with before downloading. This makes it easy to find a great looking theme for your Turbo project.

Remember, you can always change the colors, fonts, and other attributes in your CSS to make the theme more customized for your project.
